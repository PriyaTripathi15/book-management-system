
# Book Management System

A React + Vite + Tailwind application for managing books with full CRUD operations using a REST API.

## Assignment Coverage

This app includes:

- View all books
- Add a new book
- Edit/update an existing book
- Delete a book
- Search by title or author
- Filter by genre
- Loading states (fetch + save/delete)
- Basic API error handling

## Tech Stack

- React 18
- Vite
- Tailwind CSS
- Axios
- Mock API provider (MockAPI / JSON Server / equivalent)

## Project Structure

- `src/App.jsx`: State orchestration and app flow
- `src/components/BookForm.jsx`: Add/update form
- `src/components/SearchBar.jsx`: Search and genre filter
- `src/components/BookCard.jsx`: Book display with edit/delete actions
- `src/services/booksApi.js`: API setup and CRUD methods

## API Contract

Resource: `/books`

Book object fields:

- `id` (string, generated by API)
- `title` (string)
- `author` (string)
- `genre` (string)
- `year` (string or number)

## Local Setup

1. Install dependencies:

```bash
npm install
```

2. Create your local env file:

```powershell
Copy-Item .env.example .env
```

3. Update `.env` values:

```env
VITE_API_BASE_URL=https://your-project.mockapi.io/api/v1
VITE_API_KEY=
```

Notes:

- `VITE_API_BASE_URL` can be either a base URL (ending in `/api/v1`) or the full books endpoint (ending in `/books`).
- `VITE_API_KEY` is optional and only needed for providers that require API-key headers.

4. Start the app:

```bash
npm run dev
```

5. Create production build:

```bash
npm run build
```

## API Provider Setup (MockAPI)

1. Create project at https://mockapi.io
2. Create resource: `books`
3. Add fields: `title`, `author`, `genre`, `year`
4. Copy the base URL and set `VITE_API_BASE_URL` in `.env`

If requests fail with 404, verify that:

- You created `books` resource
- Your URL includes `/api/v1` when required by your provider

## Deployment

### Vercel

1. Push code to GitHub
2. Import repo in Vercel
3. Add environment variables in Vercel project settings:
   - `VITE_API_BASE_URL`
   - `VITE_API_KEY` (only if required)
4. Deploy

### Netlify

1. Push code to GitHub
2. Import repo in Netlify
3. Build command: `npm run build`
4. Publish directory: `dist`
5. Add environment variables:
   - `VITE_API_BASE_URL`
   - `VITE_API_KEY` (only if required)
6. Deploy

## Protecting API Keys

- `.env` is ignored by git via `.gitignore`.
- Only commit `.env.example`.
- Never hardcode keys in source files.
- Configure secrets in deployment platform environment settings.
